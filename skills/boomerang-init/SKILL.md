# Boomerang Init

## Description

Initialize and personalize Boomerang agents for your project by downloading canonical agent definitions from GitHub and appending project-specific personalization.

**Usage**: `/boomerang-init`

## Core Workflow

### 1. Download Canonical Agents from GitHub

Agents are fetched from the canonical source:
```
https://raw.githubusercontent.com/Veedubin/boomerang-v2/main/agents/{AGENT_NAME}.md
```

Download each agent file to `.opencode/agents/{AGENT_NAME}.md`.

**Agent list to download:**
- boomerang.md
- boomerang-coder.md
- boomerang-explorer.md
- boomerang-git.md
- boomerang-architect.md
- boomerang-tester.md
- boomerang-writer.md
- boomerang-linter.md
- boomerang-scraper.md
- boomerang-handoff.md
- researcher.md

### 2. Analyze Project

Examine the project to understand:
- **Language/Stack**: package.json, Cargo.toml, go.mod, pyproject.toml, etc.
- **Purpose**: README.md, project description
- **Conventions**: coding standards, naming patterns, architecture preferences
- **Stakeholders**: Who uses this project
- **Success metrics**: What makes this project successful

### 3. Generate Personalization Content

Create project-specific context including:
- Domain description
- Coding conventions and patterns
- Important architectural decisions
- Tool preferences specific to this project
- Success definitions

### 4. Append Personalization Section

Add personalization to each agent file using the format below.

## Personalization Section Format

Append after a `---` separator at the end of each file:

```markdown
---

## PROJECT CONTEXT
<!-- PERSONALIZATION_SECTION_START -->
<!-- Auto-generated by boomerang-init - DO NOT MODIFY ABOVE THIS LINE -->

[Project-specific content goes here]

<!-- PERSONALIZATION_SECTION_END -->
```

**Key rules:**
- The section is append-only (never modifies content above the separator)
- Contains project-specific context that tailors the agent
- Isolated between clearly marked comment markers
- Can be regenerated by re-running boomerang-init

## Execution Steps

```
1. Create .opencode/agents/ directory if it doesn't exist
2. Download each canonical agent from GitHub
3. Analyze project structure and detect stack/purpose
4. Generate personalization content based on analysis
5. Append personalization section to each downloaded agent
6. Report what was done
```

## Project Analysis Checklist

Examine these files:
- [ ] package.json / pyproject.toml / Cargo.toml / go.mod (language/framework)
- [ ] README.md (project purpose)
- [ ] src/ or lib/ directory structure (main domains)
- [ ] .github/workflows/ (CI/CD, deployment patterns)
- [ ] .opencode/ or .vscode/ (existing agent configs)
- [ ] AGENTS.md (if exists - shows current agent setup)

Detect:
- **Language**: JavaScript/TypeScript, Python, Rust, Go, etc.
- **Framework**: React, FastAPI, Express, etc.
- **Project type**: Web app, library, CLI tool, ML project, etc.
- **Special requirements**: Database, API integrations, specific tooling

## Personalization Content Template

```markdown
### Domain
[One sentence describing what this project does]

### Tech Stack
- Language: [language]
- Framework: [framework if applicable]
- Key Libraries: [important dependencies]

### Conventions
- [Convention 1 - e.g., "Use PascalCase for React components"]
- [Convention 2 - e.g., "Prefer async/await over callbacks"]

### Code Style
- [Style preference 1]
- [Style preference 2]

### Success Metrics
- [What "good code" looks like in this project]
- [Key quality signals]
```

## Output Format

After completing, report:

```
## Boomerang Init Complete

### Agents Downloaded:
- [list of agent files from GitHub]

### Agents Personalized:
- [list of agents that received project context]

### Personalization Applied:
- Domain: [project domain]
- Stack: [language/framework]
- Conventions: [count] items added

### Next Steps:
1. Review personalized agents in .opencode/agents/
2. Adjust personalization if needed
3. Start working with your agents!
```

## Idempotency

Running boomerang-init multiple times should:
- Re-download canonical agents (ensuring you have latest)
- Regenerate personalization content from current project state
- Preserve any manual customizations below the personalization markers

## File Modifications

This skill creates/modifies:
- `.opencode/agents/*.md` - Downloaded agent definitions with personalization appended

It does NOT modify:
- Source code files
- Configuration files
- Existing project files outside .opencode/

## Forbidden Operations

Never:
- Delete agent files
- Modify the core agent logic above the `---` separator
- Change agent routing or protocol logic
- Remove quality gates or mandatory steps
